# About

This framework was built with some opinions in mind and aimed to solve a few problems.

What's the benefit of being classless?

In my experience, a lot of projects kick off without ever having the most basic building blocks of the site or app taken care of. A site cannot be entirely classless, but this is a great starting point.

What's with the data attributes?

Building with data attributes instead of classes for basic frameworks serve two purposes:

  1. Prove direct access to the data attribute values in CSS.
  2. Easily setup your website for a clean JavaScript implementation with best practices in mind.
Isn't using data attributes less performant?

While there has been some evidence to suggest that data attributes is less performance than classnames, we're using them here sparingly and browser performance in the last five years has improve so much that the difference in performance is negligable.

Why doesn't tabbing work in Safari?

By default keyboard accessibility is turned off. To enable it for Safari you must open preferences and enable "Press Tab to highlight each item on a webpage".

Why don't some features work?

This framework was built to attempt to accommodate as many browser features as possible. I'd suggest looking at Can I Use to make sure that you have browser support for a feature.
